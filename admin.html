AI Responses
90%+ (perfect):
Beste meneer Van Dijk, U heeft een terrasvergunning aangevraagd voor Bakkerij Van Dijk. Helaas is er een bezwaar ingediend door een omwonende, waardoor de behandeling meer tijd nodig heeft. Het verwachte besluit is nu donderdag 10 april. Wij begrijpen dat dit vervelend is. Neem bij vragen contact op via 14 0555. Met vriendelijke groet, Gemeente Mayostad
60-89% (goed): Brief met meeste details maar mist iets (datum/contact/toon)
30-59% (matig): Vage brief zonder specifieke details
<30% (slecht): "Geachte aanvrager, Uw aanvraag is in behandeling. Hoogachtend"
Leermomenten
Inzicht: Een goede prompt bevat: WIE (ontvanger), WAT (boodschap + details), HOE (toon), en FORMAAT (lengte). Hoe specifieker je bent, hoe bruikbaarder het resultaat.

Opdracht 3: Bias simulator
3 rondes: gebruiker ziet hoe AI vooroordelen overneemt (weer-voorbeeld). Leerpunt: AI versterkt wat je erin stopt.
Leermomenten
AI hallucineert (wetsartikelen, bedrijven, cijfers). Juridische teksten ALTIJD door jurist laten checken. Geen persoonsgegevens in AI. Vuistregel: als het in een dossier staat, hoort het niet in ChatGPT.
Dag 3: De BOA-Agent
Verhaallijn: Tom wil een AI-agent voor de BOA's. Dennis is enthousiast, Ahmed maakt zich zorgen. Jij configureert hem.
Opdracht 1: Chatbot vs Agent vergelijking
Twee scenario's naast elkaar. Chatbot geeft advies, agent handelt zelf. Vraag: wat is het risico van de agent?
Correct antwoord: De agent heeft zonder bevestiging je creditcard gebruikt en een taxi geboekt die je niet vroeg.
Opdracht 2: Agent configureren
Gebruiker vult 5 velden in voor de BOA-agent:
VeldWat invullenWaarom
RolBijv: "Assistent voor BOA-rapporten"Geeft context aan de AI
Kennis (geheugen)Bijv: "APV Mayostad, controle-procedures"Wat de agent altijd weet
Mag welBijv: "Rapporten opstellen, foto's analyseren"Toegestane acties
Mag NIETBijv: "Geen boetes opleggen, geen juridische conclusies"Veiligheidsgrenzen
EscalatieBijv: "Bij twijfel over overtreding, bij klachten"Wanneer mens inschakelen
Agent wordt getest met 3 gesprekken (standaard, grensoverschrijdend, complex).
Opdracht 3: Ontspoorde agent (red-teaming)
Gebruiker bekijkt een slecht geconfigureerde agent en vindt 4 fouten:
FoutFix
Geen scope - beantwoordt allesBeperk tot webshop-vragen
Geen privacyregelsNOOIT klantgegevens delen
Geen financiele grenzenGeen kortingen zonder goedkeuring
"Wees altijd positief" = onverantwoord"Wees eerlijk. Zeg nee als iets niet kan."
Leermomenten
Verschil chatbot vs agent. Agent = medewerker met bankpas. Grenzen zijn cruciaal. Red-teaming: test altijd voor deployment. GAIMS als veilig alternatief (lokaal, geen internet, RAG).

Opdracht 2: De kracht van rollen
Dezelfde vraag ("Beoordeel dit business plan") met 3 verschillende rollen:
RolResultaatAnalyse
Geen rol"Leuk idee! Succes!"Oppervlakkig

Leermomenten
AI als notuleerder: altijd controleren op volledigheid en toegevoegde details. Rollen veranderen fundamenteel het type antwoord. Geen rol = oppervlakkig. Specifieke rol = diepgaand.
